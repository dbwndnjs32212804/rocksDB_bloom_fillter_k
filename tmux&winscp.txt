1. WinSCP (Windows Secure Copy)
Windows 환경에서 원격 리눅스 서버(Ubuntu)로 파일을 안전하게 전송하거나, 서버 내부의 파일을 로컬 에디터처럼 직접 수정할 수 있게 해주는 그래픽 기반의 SFTP/FTP 클라이언트

주요 특징
GUI 기반 편집: 리눅스의 vi나 nano 같은 터미널 에디터가 익숙하지 않아도, Windows의 메모장이나 VS Code를 연동하여 서버 파일을 시각적으로 편집할 수 있다 
드래그 앤 드롭: 로컬과 서버 간의 파일 복사 및 이동이 매우 직관적이다.
동기화 기능: 로컬 폴더와 서버 폴더를 동기화하여 변경된 코드만 자동으로 업로드할 수 있다.

장점
코드 수정(bloom_impl.h 등) 시 전체 구조를 한눈에 파악하며 작업하기 편리하다. 
파일 업로드/다운로드 속도가 빠르고 직관적이다.


2. tmux (Terminal Multiplexer)
단일 터미널 세션에서 여러 가상 세션을 생성하고 관리하는 도구로, 원격 접속이 끊겨도 프로세스를 유지하는 '세션 지속성'이 핵심인 도구

주요 특징
세션 유지(Persistence): 인터넷 연결이 끊겨도 서버 내부의 db_bench 실험은 멈추지 않고 계속 돌아간다. 
화면 분할: 한 세션 내에서 화면을 나누어 실험 실행과 리소스 모니터링을 동시에 할 수 있다.

장점
오랜 시간이 걸리는 대규모 실험(num=10,000,000 등)을 안심하고 수행할 수 있다. 
서버 재접속 시 이전 작업 상태를 그대로 복구(Attach)할 수 있다.


3. WinSCP와 tmux의 결합 (효율적인 원격 실험 워크플로우)
두 도구를 함께 사용하면 효율적인 실험 파이프라인이 완성된다.
WinSCP (편집): Windows 환경에서 GUI 에디터를 사용해 bloom_impl.h의 k값을 편리하게 수정한다. 
tmux (실행): 수정된 코드를 컴파일(make)하고, db_bench 명령어를 실행한다. 
중간에 컴퓨터를 끄거나 이동하더라도 실험은 서버에서 안전하게 계속된다. 
WinSCP (확인): 실험 결과 파일(result.txt)을 WinSCP로 다시 열어보거나 로컬로 가져와 데이터 분석을 진행한다.
